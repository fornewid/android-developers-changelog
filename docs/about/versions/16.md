# Android 16

### [![Android 16](https://developer.android.com/static/images/about/versions/16/android-16-wordmark.png)](http://developer.android.com/about/versions/16/get)

Android 16 continues our mission of building a private and secure platform that helps improve your productivity while giving you new capabilities to produce beautiful apps, superior media and camera experiences, and an intuitive user experience, particularly on tablets and foldables.  
[Get started](http://developer.android.com/about/versions/16/get)[Test your app](http://developer.android.com/about/versions/16/get#on_emulator)

## Get started with Android 16

1. **Set up a runtime environment** --- see[Get Android 16](https://developer.android.com/about/versions/16/get)to flash a Google Pixel device or set up an emulator.
2. **Set up Android Studio** --- try the Android 16 SDK and tools. See the[SDK setup](https://developer.android.com/about/versions/16/setup-sdk)page for steps.
3. **Learn about what's new** --- review the[behavior changes for all apps](https://developer.android.com/about/versions/16/behavior-changes-all)and the[behavior changes for apps targeting Android 16](https://developer.android.com/about/versions/16/behavior-changes-16)that might've affected your app.
4. **Test your app** --- run through all flows to look for issues.[Toggle behavior changes](https://developer.android.com/about/versions/16/reference/compat-framework-changes)at runtime to isolate issues.
5. **Update your app**--- target Android 16 if possible, and test with users using beta channels or other groups.
