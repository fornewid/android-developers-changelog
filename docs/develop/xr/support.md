# Get support for the Android XR SDK Stay organized with collections Save and categorize content based on your preferences.



Thanks for your interest in the Developer Preview of the Android XR SDK. In
addition to the overall [Android Developer Support Resources](/support), use the links
on this page to view release notes, search open issues, or file new issues
related to the Android XR SDK. We want to hear from you, and we welcome your
feedback.

## Determine where to file your issue

To get the fastest resolution, file your issue with the team best equipped to
handle it. Review the following list to determine the appropriate support
channel:

* **Game engine and tooling issues**: If you encounter bugs with a game engine
  editor, standard OpenXR features, or engine-specific tools, please report
  them to the engine provider ([Unity](#unity), [Godot Foundation](#godot), or [Epic
  Games](#unreal)).
* **Android XR platform and OS issues**: If you encounter bugs specific to the
  Android XR operating system, system behavior, or device compatibility (that
  **do not** reproduce on other OpenXR devices), please report them to Google
  using the channels described on this page in the [report issues](#report-issues) section.

## Include detailed information in bug reports

When reporting bugs, make sure you include the following information in your bug
description:

* A clear and concise description of what the bug is.
* Steps to reproduce the behavior.
* A clear and concise description of what you expected to happen.
* A clear and concise description of what you observed.
* [Capture](/studio/debug/bug-report) a bug report and share the Google Drive link. If the bug report
  contains sensitive information, then make it private and only give access to
  requests from @google.com accounts.
* Add screenshots to help explain your problem if applicable.
* To learn how to use the Android logging system, see the [Logcat command-line
  tool](/tools/debugging/debugging-log) documentation.

## Get support for game engines

See each of the following sections for details about getting support while
developing for Android XR using game engines.

### Android XR Extensions for Unity

Use these links for issues related to the Android XR Extensions for Unity
package or the included samples.

* [Report an issue](https://github.com/android/android-xr-unity-package/issues/new)
* [Search issues](https://github.com/android/android-xr-unity-package/issues)
* [Release notes](https://github.com/android/android-xr-unity-package/releases)

### Godot Engine

Support for Android XR is available using the OpenXR Vendors plugin.

* **Engine issues**: As an open-source project, Godot support is handled
  through [**GitHub Issues**](https://github.com/godotengine/godot/issues) and community channels (Discord
  and forums).
* **Documentation**: [Godot XR Documentation](https://docs.godotengine.org/en/stable/tutorials/xr/index.html)
* **Android XR platform issues**: If an issue is specific only to Android XR
  (and does not reproduce on other OpenXR devices), report it using the links
  referenced in the preceding [report issues](#report-issues) section.

### Unreal Engine

If you encounter issues, please use the Unreal Engine XR Development Forum. The
Unreal Engine XR team actively monitors this forum.

* **Engine and tooling issues**: Post your questions or bug reports in the
  [Unreal Engine XR Development Forum](https://forums.unrealengine.com/tags/c/development-discussion/platforms/xr-development/32/unreal-engine).
* **Documentation**: [Unreal XR Development Guide](https://dev.epicgames.com/documentation/en-us/unreal-engine/developing-for-head-mounted-experiences-with-openxr-in-unreal-engine)
* **Android XR platform issues**: If an issue is specific only to Android XR
  (and does not reproduce on other OpenXR devices), report it using the links
  referenced in the preceding [report issues](#report-issues) section.
