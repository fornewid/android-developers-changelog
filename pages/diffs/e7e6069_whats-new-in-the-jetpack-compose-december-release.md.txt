 ## **New features**
 
 ### **Retain**
 
-Compose offers a number of APIs to hold and manage state across different lifecycles; for example, `remember` persists state across compositions, and `rememberSavable`/`rememberSerializable` to persist across activity or process recreation. [`retain`](/reference/kotlin/androidx/compose/runtime/retain/package-summary#retain(kotlin.Function0)) is a new API that sits between these APIs, enabling you to persist values across configuration changes without being serialized, but not across process death. As `retain` does not serialize your state, you can persist objects like lambda expressions, flows, and large objects like bitmaps, which cannot be easily serialized.
